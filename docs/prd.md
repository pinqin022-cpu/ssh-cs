一、产品核心功能概述
本小程序核心为单人专属抽奖工具，用户可自定义格子式抽奖模板，生成唯一分享链接，仅首个点击链接的用户可参与抽奖，发起人可管理抽奖活动，参与者可查看参与记录。
二、页面结构与交互逻辑
1. 全局导航结构
页面位置        元素名称        功能说明
首页核心区        发起抽奖按钮        点击进入抽奖创建页面，为核心功能入口
底部 Tab 栏        首页        点击返回小程序首页（默认展示页）
底部 Tab 栏        我的        点击进入个人中心页面
其他页面必左上角必须要有返回键回到首页和我的
2. 核心页面详细规则
（1）抽奖创建页（点击“发起抽奖”进入）
基础配置：
支持自定义抽奖格子行列数（如3×3、4×4），默认选中3×3模板；
强制输入“抽奖名称”（必填项，不能为空），该名称会展示在抽奖参与页顶部。
奖品/内容配置：
每个格子为独立编辑项，发起人需填写每个格子对应的奖品/内容文字（支持空值，代表啥都没有”）；
所有格子填写完成后，点击“确认创建”按钮，触发弹窗（文案：“抽奖创建成功！”），弹窗包含“关闭”和“分享”按钮。
链接生成规则：
确认创建后自动生成唯一分享链接，链接与当前抽奖活动一一绑定；
链接参与规则：仅第一个点击链接的用户可参与抽奖，后续用户点击链接时弹窗提示“该抽奖链接已被他人使用，无法参与”。
（2）抽奖参与页（点击分享链接进入）
视觉展示：
顶部展示抽奖名称，下方展示格子布局（未抽奖时所有格子为默认红色）；
页面仅展示“开始抽奖”按钮（格子下方），点击后触发随机抽奖逻辑。
随机抽奖规则：
采用真实随机算法（如基于时间戳+用户ID的哈希值取模，避免伪随机），随机选中一个未开奖格子；
抽奖结果展示：
抽中后弹窗展示结果（文案：“恭喜你抽中：【XXXXX】”，XXXXX为该格子填写的内容）；
弹窗底部有“关闭”和“分享抽奖结果”按钮，点击“分享”可将抽中结果转发给其他用户；
视觉更新：抽中格子变为灰色，标注“已开奖”（所有人可见）；未抽中格子统一显示为红色，内容不可见（仅发起人可在管理页查看）。
冷却时间规则：
发起人可在创建页设置“抽奖冷却时间”（如24小时），规则为：同一用户在冷却时间内仅能参与1次该发起人创建的抽奖（基于用户小程序OpenID判断）；
若用户在冷却时间内重复点击“开始抽奖”，弹窗提示“你已在XX小时内参与过该发起人的抽奖，请稍后再试”。
全部开奖规则：
当所有格子均被抽中（已开奖），用户点击“开始抽奖”时弹窗提示：“抽奖已结束”。
（3）“我的”页面（个人中心）
核心模块：
模块1：我发起的抽奖
展示列表：包含所有发起人创建的抽奖活动，每项展示信息：抽奖名称、发起时间、状态标签（绿色=进行中，灰色=已结束（所有格子开奖/链接已被使用））；
操作权限：点击发起人创建的抽奖活动可进入“抽奖管理页”，支持：
修改未开奖格子的内容；
修改抽奖冷却时间；
重新生成分享链接（修改内容后自动生成新链接，旧链接失效）。
模块2：开奖记录管理（仅发起人可见）
展示列表：包含该发起人创建的所有抽奖的开奖记录，每项展示：抽奖名称、抽奖时间、抽中格子的内容；
支持按时间倒序排列（最新开奖在前）。
模块3：我参与的抽奖（仅参与者可见）
展示列表：包含该用户参与过的所有抽奖活动，每项展示：抽奖名称、抽奖时间、抽中内容
支持按时间倒序排列。

四、补充业务规则（开发需注意）
数据存储：需记录每个抽奖活动的基础信息（名称、格子数、内容、创建时间、状态）、开奖记录（用户ID、抽奖时间、抽中格子）、链接使用状态（是否被使用、使用用户ID）；
权限控制：发起人仅能管理自己创建的抽奖，参与者仅能查看自己的参与记录；
异常处理：网络异常时提示“加载失败，请重试”，链接失效/已开奖时明确提示原因。
五、界面和使用感觉要求
* 界面样子:
    * 必须使用 TDesign Weixin 这个现成的界面工具包来做，让小程序的按钮、列表、输入框等看起来风格统一、简洁、好看。 (参考: `https://tdesign.tencent.com/miniprogram/getting-started`)

* 使用感觉:
    * 用起来要流畅，滑动列表、切换页面不能卡。
    * 点了按钮、等图片生成、加载更多图片时，都要有提示，让用户知道现在是什么状态。
* 速度:
    * 小程序打开速度要快。
    * 看图片（尤其首页的很多图）加载要快。
* 安全:
    * 微信登录过程要安全，不能泄露用户信息。
